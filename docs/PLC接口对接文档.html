典型操作时序

上电流程

sequenceDiagram participant PLC participant pn_dev participant Robot Note over PLC,Robot: 初始状态: BOOL[0]=0, BOOL[2]=0 PLC->>pn_dev: PROFINET: BOOL[0] = 1 (上升沿) pn_dev->>Robot: TCP: PLCFrame (BOOL[0]=1) Robot->>Robot: handlePLCData() 检测上升沿 Robot->>Robot: gpio_safety_start() Robot->>pn_dev: TCP: status (BOOL[0]=1, BOOL[2]=1) pn_dev->>PLC: PROFINET: 电源=1, 使能=1 Note over PLC: PLC 确认: BOOL[0]=1 AND BOOL[2]=1 表示就绪 PLC->>pn_dev: PROFINET: BOOL[0] = 0 (复位) pn_dev->>Robot: TCP: PLCFrame (BOOL[0]=0)

急停流程

sequenceDiagram participant PLC participant pn_dev participant Robot Note over PLC,Robot: 初始状态: 正常运行，BOOL[6]=1 (Busy) PLC->>pn_dev: PROFINET: BOOL[4] = 1 (上升沿) pn_dev->>Robot: TCP: PLCFrame (BOOL[4]=1) Robot->>Robot: handlePLCData() 检测上升沿 Robot->>Robot: triggerStop() Robot->>pn_dev: TCP: status (BOOL[6]=0, BOOL[1]=1) pn_dev->>PLC: PROFINET: Busy=0, 告警=1 Note over PLC: PLC 确认: BOOL[6]=0 AND BOOL[1]=1 表示已停止 PLC->>pn_dev: PROFINET: BOOL[4] = 0 (复位)

与 .proc 程序协作

sequenceDiagram participant PLC participant pn_dev participant Robot participant Proc as .proc 程序 Note over PLC,Proc: PLC 通过程序变量区传递命令参数 PLC->>pn_dev: REAL[0~5] = 目标数据 PLC->>pn_dev: INT[0] = 命令码 PLC->>pn_dev: BOOL[8] = 1 (触发位 ↑) pn_dev->>Robot: TCP 透传 Robot->>Proc: 同步程序变量 Proc->>Proc: 解析命令码 + 参数 Proc->>Proc: 执行运动指令 Robot->>pn_dev: BOOL[6]=1 (Busy) pn_dev->>PLC: Busy=1 Proc->>Proc: 运动完成 Robot->>pn_dev: BOOL[6]=0, BOOL[16]=1 (Done, 保持300ms) pn_dev->>PLC: Done=1 Note over PLC: PLC 检测 BOOL[16] 上升沿 → 锁存完成信号 PLC->>pn_dev: BOOL[8] = 0 (触发位复位)
PLC 编程提示：
1. BOOL[16] 仅保持 300ms，PLC 必须检测上升沿后立即锁存
2. 触发位 (如 BOOL[8]) 需要在检测到 Done 后复位，为下次触发做准备
3. 命令码和参数的含义由 .proc 程序定义，PLCFrame 本身不包含运动语义

注意事项

#事项
1BOOL[0] 和 BOOL[4] 是上升沿触发，操作后必须复位为 0，否则无法再次触发
2BOOL[16] 仅保持 300ms，PLC 应检测上升沿后立即锁存
3REAL 是 IEEE 754 单精度浮点，注意字节序（Big-Endian 传输）
4运动指令的具体含义由 .proc 程序定义，PLCFrame 本身不直接包含运动语义
5未使用的区域保持为 0
6GSDML 文件名：GSDML-V2.43-RT-Labs-P-Net-Sample-App-20240530.xml
