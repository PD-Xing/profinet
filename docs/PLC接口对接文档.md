# PLC 侧数据格式说明

## 1. PROFINET 传输参数

pn_dev 设备以 80 字节 I/O 模块与 PLC 交换数据。

| 参数 | 值 |
|------|-----|
| 设备名称 | `rt-labs-dev` |
| 输出数据（PLC→设备） | **80 字节**，周期 1ms |
| 输入数据（设备→PLC） | **80 字节**，周期 1ms |
| GSDML 模块 | PLC Frame（Ident 0x00000050） |

---

## 2. 80 字节数据结构

```
字节偏移  0                              32                   48                  80
         ├───────────────────────────────┼─────────────────────┼───────────────────┤
         │      BOOL[256] (32字节)        │  INT[8] (16字节)     │  REAL[8] (32字节)  │
         ├───────────────────────────────┴─────────────────────┴───────────────────┤
         │                          合计 80 字节                                    │
         └─────────────────────────────────────────────────────────────────────────┘
```

**PLC 侧地址映射**（假设起始地址为 0）：

| 区域 | 字节偏移 | PLC 地址 | 类型 |
|------|---------|---------|------|
| BOOL[0~255] | 0~31 | %QB0 ~ %QB31 | 32 个 BYTE |
| INT[0~7] | 32~47 | %QW32 ~ %QW46 | 8 个 INT |
| REAL[0~7] | 48~79 | %QD48 ~ %QD76 | 8 个 REAL |

---

## 3. PLC → 设备（输出，Output）

### 3.1 硬件控制（边缘触发）

| 地址 | 信号 | 触发方式 | 功能 |
|------|------|---------|------|
| %QB0.0 | BOOL[0] | **上升沿** | 上电启动（调用 gpio_safety_start） |
| %QB0.4 | BOOL[4] | **上升沿** | 软急停（调用 triggerStop） |

### 3.2 视觉偏移（连续值，实时生效）

| 地址 | 信号 | 功能 |
|------|------|------|
| %QD48 | REAL[0] | 视觉引导 X 方向偏移 |
| %QD52 | REAL[1] | 视觉引导 Y 方向偏移 |
| %QD56 | REAL[2] | 视觉引导 Z 方向偏移 |
| %QD60 | REAL[3] | 视觉引导姿态偏移 |

### 3.3 程序变量区（供 .proc 程序引用）

以下所有数据会被同步到机器人的程序变量，`.proc` 程序中通过 `DATA_BOOL[N]`、`DATA_INT[N]`、`DATA_REAL[N]` 引用：

| 地址 | 变量 | 用途 |
|------|------|------|
| %QB0 ~ %QB31 | DATA_BOOL[0~255] | 256 个布尔变量 |
| %QW32 ~ %QW46 | DATA_INT[0~7] | 8 个有符号整数变量 |
| %QD48 ~ %QD76 | DATA_REAL[0~7] | 8 个浮点变量 |

> **说明**：REAL[0~3] 有双重用途——既进入程序变量，又被硬件用作视觉偏移。其余 REAL[4~7] 仅进入程序变量。

---

## 4. 设备 → PLC（输入，Input）

### 4.1 状态信号

| 地址 | 信号 | 代码来源 | 说明 |
|------|------|---------|------|
| %IB0.0 | BOOL[0] | gpio_safety_get_state()==RUNNING | 电源已上电 |
| %IB0.1 | BOOL[1] | EnableStatus==0 | 告警（电机未使能） |
| %IB0.2 | BOOL[2] | EnableStatus!=0 | 已使能 |
| %IB0.6 | BOOL[6] | trajectory_cache_.is_active | 轨迹执行中 |
| %IB0.7 | BOOL[7] | sum\|qd\| > 0.005 | 关节运动中 |
| %IB1.7 | BOOL[15] | sum\|qd\| > 0.005 | 动作忙（同 Moving） |
| %IB2.0 | BOOL[16] | now < done_expire | 动作完成（保持 300ms） |

### 4.2 夹爪

| 地址 | 信号 | 代码来源 | 说明 |
|------|------|---------|------|
| %IB2.1 | BOOL[17] | tool_realtime_.ai0 > 128 | 左夹爪闭合 |
| %IB2.2 | BOOL[18] | tool_realtime_.ai1 > 128 | 右夹爪闭合 |
| %IB4.0 | BOOL[32] | tool_realtime_.di bit0 | 左夹爪有料 |
| %IB4.1 | BOOL[33] | tool_realtime_.di bit1 | 右夹爪有料 |

### 4.3 板载传感器

| 地址 | 信号 | 说明 |
|------|------|------|
| %IB4.2 | BOOL[34] | DI bit0 |
| %IB4.3 | BOOL[35] | DI bit1 |
| %IB4.4 | BOOL[36] | DI bit2 |
| %IB4.5 | BOOL[37] | DI bit3 |

### 4.4 int_data 和 real_data（输入方向）

| 地址 | 状态 |
|------|------|
| %IW32 ~ %IW46（INT[0~7]） | 当前全零，预留给后续扩展 |
| %ID48 ~ %ID76（REAL[0~7]） | 当前全零，预留给后续扩展 |

## 5. 典型操作流程

### 上电
```
PLC 写: BOOL[0] 设为 1（上升沿触发）
PLC 读: BOOL[0]=1 且 BOOL[2]=1 表示就绪
```

### 急停
```
PLC 写: BOOL[4] 设为 1（上升沿触发）
PLC 读: BOOL[6]=0（不再 Busy），BOOL[1]=1（告警）表示已停止
```

### 与 .proc 程序协作

运动指令的执行逻辑在机器人侧的 `.proc` 程序中实现，PLC 通过程序变量区传递参数：

```
PLC 写: REAL[0~5] = 目标数据（含义由 .proc 程序定义）
PLC 写: INT[0] = 命令码（含义由 .proc 程序定义）
PLC 写: 触发位（如 BOOL[8]）设为 1
PLC 读: 等待 BOOL[6]=1 → BOOL[16]=1（动作完成）
PLC 写: 触发位复位为 0
```

---

## 6. 数据采样示意（PLC 程序参考）

```
// === 输出（PLC→设备）===

"DB_Frame".Output.bool_data[0].%X0 := "上电按钮";     // BOOL[0]
"DB_Frame".Output.bool_data[0].%X4 := "急停按钮";     // BOOL[4]
"DB_Frame".Output.real_data[0] := "视觉偏移X";        // REAL[0]

// === 输入（设备→PLC）===

"设备已上电"  := "DB_Frame".Input.bool_data[0].%X0;   // BOOL[0]
"设备告警"    := "DB_Frame".Input.bool_data[0].%X1;   // BOOL[1]
"设备已使能"  := "DB_Frame".Input.bool_data[0].%X2;   // BOOL[2]
"轨迹执行中"  := "DB_Frame".Input.bool_data[0].%X6;   // BOOL[6]
"运动中"      := "DB_Frame".Input.bool_data[0].%X7;   // BOOL[7]
"动作完成"    := "DB_Frame".Input.bool_data[2].%X0;   // BOOL[16]
```

---

## 7. 注意事项

1. **BOOL[0] 和 BOOL[4] 是上升沿触发**，需操作后复位为 0
2. **BOOL[16] 仅保持 300ms**，PLC 应检测上升沿后立即锁存
3. **REAL 是 IEEE 754 单精度浮点**
4. **运动指令的具体含义由 .proc 程序定义**，PLCFrame 本身不直接包含运动语义
5. 未使用的区域保持为 0
