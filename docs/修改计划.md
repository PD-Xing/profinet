# pn_dev 接入运动控制器 — 修改计划

## 目标

pn_dev 作为桥梁，将 PLC 通过 PROFINET 发来的 80 字节 PLCFrame 透传给运动控制器（TCP 127.0.0.1:31000），并将运动控制器的状态回传给 PLC。

## 现有代码架构

```
PROFINET 1ms tick
  ↓
sampleapp_common.c: app_cyclic_data_callback()
  ↓
pnet_output_get_data_and_iops()  → 取出 PLC 发来的字节
  ↓
app_data.c: app_data_set_output_data(data, size)  ← 当前：解析位、串口打印
  ↓
app_data.c: app_data_get_input_data(...)          ← 当前：返回本地 1 字节
  ↓
pnet_input_set_data_and_iops() → 发回给 PLC
```

## 为什么需要修改

| 文件 | 当前状态 | 问题 | 需要变成 |
|------|---------|------|---------|
| `app_gsdml.h` | 最大模块 8 字节（Echo） | 装不下 80 字节 PLCFrame | 新增大模块常量 |
| `app_gsdml.c` | 模块目录缺 PLC Frame | PROFINET 无法协商大模块 | 注册新模块到目录 |
| GSDML XML | 只有 DIO/Echo 模块 | TIA Portal 看不到 80 字节模块 | 新增模块定义 |
| `app_data.c` | 1 字节位解析 + 串口 | 无 TCP 桥接 | TCP 客户端 + PLCFrame 透传 |
| `sampleapp_main.c` | 调用 app_data_init() | TCP 线程需要在此启动 | 加 TCP 启动调用 |

其余文件（`app_utils.c/h`、`app_log.c/h`、`sampleapp_common.c/h`、p-net 协议栈、OSAL）**不需要修改**。

## 修改清单

### 修改 1：app_gsdml.h — 新增 PLC Frame 常量

**文件**：`~/profinet/p-net-1.0.2-samples/pn_dev/app_gsdml.h`

**为什么改**：GSDML 模块目录通过 `#define` 常量定义 Ident Number，
PROFINET 协议在建立连接时用这些 ID 协商模块类型。
没有这组常量，PLC 就无法识别 80 字节的 PLC Frame 模块。

**改动**：在文件末尾（`#endif` 之前）新增：

```c
/* PLC Frame module (80 bytes) for motion controller communication */
#define APP_GSDML_MOD_ID_PLC_FRAME          0x00000050
#define APP_GSDML_SUBMOD_ID_PLC_FRAME_IN    0x00000150
#define APP_GSDML_SUBMOD_ID_PLC_FRAME_OUT   0x00000151
#define APP_GSDML_PLC_FRAME_SIZE            80
```

### 修改 2：app_gsdml.c — 注册 PLC Frame 模块

**文件**：`~/profinet/p-net-1.0.2-samples/pn_dev/app_gsdml.c`

```c
   .api = APP_GSDML_API,
   .data_dir = PNET_DIR_INPUT,
   .insize = APP_GSDML_PLC_FRAME_SIZE,
   .outsize = 0,
   .parameters = {0}};

/* PLC Frame submodule: Output (PLC → Robot) */
static const app_gsdml_submodule_t submod_plc_frame_out = {
   .id = APP_GSDML_SUBMOD_ID_PLC_FRAME_OUT,
   .name = "PLC Frame Output",
   .api = APP_GSDML_API,
   .data_dir = PNET_DIR_OUTPUT,
   .insize = 0,
   .outsize = APP_GSDML_PLC_FRAME_SIZE,
   .parameters = {0}};
```

**改动 2b**：新增模块配置：

```c
static const app_gsdml_module_t module_plc_frame = {
   .id = APP_GSDML_MOD_ID_PLC_FRAME,
   .name = "PLC Frame",
   .submodules = {APP_GSDML_SUBMOD_ID_PLC_FRAME_IN,
                  APP_GSDML_SUBMOD_ID_PLC_FRAME_OUT, 0}};
```

**改动 2c**：注册到模块目录（在 `app_gsdml_modules[]` 数组中追加）：

### 修改 3：app_data.c — TCP 桥接核心

**文件**：`~/profinet/p-net-1.0.2-samples/pn_dev/app_data.c`

**为什么改**：这是核心改动。当前 `app_data.c` 只处理 1 字节的位 I/O 并打印到串口。
需要新增 TCP 客户端连接到运动控制器的 31000 端口，并在两个数据回调中做 PLCFrame 透传。

**改动 3a**：新增头文件和全局变量（文件开头 `#include` 区之后）：

```c
/* TCP bridge to motion controller */
#include <sys/socket.h>
#include <netinet/in.h>
#include <arpa/inet.h>
#include <unistd.h>
#include <pthread.h>
#include <errno.h>

#define PLC_BRIDGE_HOST     "127.0.0.1"
#define PLC_BRIDGE_PORT     31000

/* PLCFrame: 80-byte protocol with motion controller */
#pragma pack(push, 1)
typedef struct {
   uint8_t  bool_data[32];
   int16_t  int_data[8];
   float    real_data[8];
} PLCFrame;
#pragma pack(pop)

static int plc_sock_fd = -1;
static PLCFrame latest_robot_status;
static pthread_mutex_t status_mutex = PTHREAD_MUTEX_INITIALIZER;
static pthread_t plc_thread_id;
static volatile bool plc_thread_running = true;
```

**改动 3b**：新增 TCP 后台线程（在 `init_serial_port()` 之后）：

```c
/**
 * TCP bridge thread: maintain connection to motion controller.
 * On connect, continuously receives robot status PLCFrame.
 */
static void * plc_tcp_thread_fn(void * arg)
{
   (void)arg;
   struct sockaddr_in addr = {
      .sin_family = AF_INET,
      .sin_port = htons(PLC_BRIDGE_PORT),
   };
   inet_pton(AF_INET, PLC_BRIDGE_HOST, &addr.sin_addr);

   while (plc_thread_running) {
      int fd = socket(AF_INET, SOCK_STREAM, 0);
      if (fd < 0) {
         sleep(1);
         continue;
      }

      if (connect(fd, (struct sockaddr *)&addr, sizeof(addr)) != 0) {
         APP_LOG_WARNING("TCP bridge: connect to %s:%d failed: %s\n",
                         PLC_BRIDGE_HOST, PLC_BRIDGE_PORT, strerror(errno));
         close(fd);
         sleep(1);
         continue;
      }

      APP_LOG_INFO("TCP bridge: connected to motion controller %s:%d\n",
                   PLC_BRIDGE_HOST, PLC_BRIDGE_PORT);

      /* Block and read robot status frames */
      while (plc_thread_running) {
         PLCFrame rx;
         ssize_t n = recv(fd, &rx, sizeof(PLCFrame), MSG_WAITALL);
         if (n != sizeof(PLCFrame))
            break;

         pthread_mutex_lock(&status_mutex);
         latest_robot_status = rx;
         pthread_mutex_unlock(&status_mutex);
      }

      APP_LOG_WARNING("TCP bridge: disconnected, reconnecting...\n");
      close(fd);
   }

   return NULL;
}

/**
 * Start TCP bridge thread. Called once at application startup.
 */
int app_plc_bridge_start(void)
{
   if (pthread_create(&plc_thread_id, NULL, plc_tcp_thread_fn, NULL) != 0) {
      APP_LOG_WARNING("TCP bridge: failed to create thread\n");
      return -1;
   }
   return 0;
}
```

**改动 3c**：修改 `app_data_set_output_data()` — 增加 PLC Frame 转发：

在函数开头新增：

```c
   /* PLC Frame Output: forward directly to motion controller via TCP */
   if (submodule_id == APP_GSDML_SUBMOD_ID_PLC_FRAME_OUT) {
      if (data == NULL || size < sizeof(PLCFrame)) {
         return -1;
      }
      if (plc_sock_fd > 0) {
         PLCFrame tx;
         memcpy(&tx, data, sizeof(PLCFrame));
         if (send(plc_sock_fd, &tx, sizeof(PLCFrame), MSG_NOSIGNAL) < 0) {
            APP_LOG_WARNING("TCP bridge: send failed\n");
         }
      }
      return 0;
   }
```

**改动 3d**：修改 `app_data_get_input_data()` — 增加 PLC Frame 返回：

在函数开头新增：

```c
   /* PLC Frame Input: return latest robot status from TCP bridge */
   if (submodule_id == APP_GSDML_SUBMOD_ID_PLC_FRAME_IN) {
      static uint8_t status_buf[sizeof(PLCFrame)];

      pthread_mutex_lock(&status_mutex);
      memcpy(status_buf, &latest_robot_status, sizeof(PLCFrame));
      pthread_mutex_unlock(&status_mutex);

      *size = sizeof(PLCFrame);

      if (plc_sock_fd > 0) {
         *iops = PNET_IOXS_GOOD;
      } else {
         *iops = PNET_IOXS_BAD;   /* Motion controller not connected */
      }
      return status_buf;
   }
```

**注意**：原有 DIO/Echo 模块的处理逻辑**保持不变**，通过 `if (submodule_id == ...)` 分支各自处理，互不干扰。

## 修改后的数据流

```
PLC (%QB0~79)
  │ PROFINET RT, 1ms
  ▼
p-net 栈 → pnet_output_get_data_and_iops()
  │
  ▼
app_data_set_output_data(submodule_id=0x151, data=80字节PLCFrame)
  │
  │  if (submodule_id == PLC_FRAME_OUT)
  │     send(plc_sock_fd, data, 80)  ──→ 127.0.0.1:31000
  │                                         │
  │                                    运动控制器 PLCCommunicator
  │                                     recv(80) → handlePLCData()
  │                                     buildRobotToPLCFrame()
  │                                     send(80) → 
  │                                         │
  │     recv(plc_sock_fd) ←──── TCP 响应 ────┘
  │     latest_robot_status = rx
  │
  ▼
app_data_get_input_data(submodule_id=0x150)
  │  return latest_robot_status (80字节)
  ▼
p-net 栈 → PROFINET 帧 → PLC (%IB0~79)
```

## 测试验证

修改完成后，按以下顺序验证：

1. **编译**：`make -j$(nproc)`，无报错
2. **运动控制器连接**：运控启动后 31000 端口监听，pn_dev 串口打印 "TCP bridge: connected"
3. **PLC 连接**：TIA Portal 中能看到设备在线，IO 数据正常（IOPS=GOOD）
4. **PLC 上电**：PLC 设 BOOL[0]=1，机器人上电，回读 BOOL[0]=1
5. **数据回环**：PLC 写 bool_data[5]=1，回读 bool_data[5] 看是否正确反映
6. **断线恢复**：拔掉网线，PLC 侧 IOPS 变 BAD；插回网线，自动恢复
