## 2. pn_dev 程序修改

### 2.1 修改文件清单

| 文件 | 修改内容 |
|------|---------|
| `pn_dev/app_data.c` | 添加串口调试输出（`/dev/ttyFIQ0`），仅在 I/O 数据变化时打印 |
| `pn_dev/app_data.h` | 添加 `app_data_init()` 函数声明 |
| `ports/linux/sampleapp_main.c` | 添加 `app_data_init()` 调用，适配 OK3576 配置 |

### 2.2 功能说明

- **I1.0~I1.7**：从站发送给 PLC 的输入状态（1 字节）
- **Q1.0~Q1.7**：PLC 发送给从站的输出状态（1 字节）
- **串口打印**：仅在 I/O 数据变化时打印，格式：
  ```
  I1.0=0,I1.1=0,...,I1.7=0 | Q1.0=1,Q1.1=0,...,Q1.7=0
  ```

### 2.3 GSDML 模块配置

| 槽位 | 模块 | 模块 ID | 说明 |
|------|------|---------|------|
| Slot 0 | Device Access Point | - | 系统保留 |
| **Slot 1** | **DIO 8xLogicLevel** | **IDM_32 (0x32)** | **8位输入 + 8位输出（组合模块）** |

| 项目 | 版本/路径 |

工具链已复制到 `~/profinet/toolchain/`，CMake 配置文件 `cmake/aarch64-linux-gnu.cmake` 已更新为使用本地工具链：

```cmake

```

## 4. 编译 pn_dev

### 4.1 一键编译

```bash
cd ~/profinet/p-net-1.0.2-samples
./build_pn_dev.sh
```

### 4.2 手动编译

```bash
cd ~/profinet/p-net-1.0.2-samples
rm -rf build && mkdir -p build && cd build

cmake .. -DCMAKE_TOOLCHAIN_FILE=../cmake/aarch64-linux-gnu.cmake
make -j$(nproc)
```

### 4.3 编译产物

```
build/pn_dev/pn_dev    # ELF 64-bit LSB executable, ARM aarch64
```

### 4.4 验证编译产物

```bash
file build/pn_dev/pn_dev
# 输出: ELF 64-bit LSB executable, ARM aarch64

readelf -d build/pn_dev/pn_dev | grep NEEDED
# 输出: libpthread.so.0, librt.so.1, libc.so.6
# （p-net 协议栈已静态链接）
```

## 5. 部署到开发板

### 5.1 开发板环境

| 项目 | 值 |
|------|-----|
| 开发板 | OK-3576-C |
| Linux 内核 | 6.1.118 RT |
| IP 地址 | 192.168.1.100 |
| 用户名/密码 | default / default |
| 调试串口 | `/dev/ttyFIQ0` |

### 5.2 上传文件

```bash
# 上传主程序
sshpass -p 'default' scp build/pn_dev/pn_dev \
  default@192.168.1.100:/home/default/profinet/

# 上传 GSDML 文件
sshpass -p 'default' scp pn_dev/GSDML*.xml \
  default@192.168.1.100:/home/default/profinet/

# 上传辅助脚本
sshpass -p 'default' scp kill_pn_dev.sh \
  default@192.168.1.100:/home/default/profinet/

# 设置权限
sshpass -p 'default' ssh default@192.168.1.100 \
  "chmod +x /home/default/profinet/pn_dev /home/default/profinet/kill_pn_dev.sh"
```

### 5.3 开发板目录结构

```
/home/default/profinet/
├── pn_dev                                    # Profinet 从站程序
├── GSDML-V2.43-RT-Labs-P-Net-Sample-App-20240530.xml
└── kill_pn_dev.sh
```

### 5.4 运行程序

```bash
# SSH 登录
sshpass -p 'default' ssh default@192.168.1.100

# 进入目录
cd /home/default/profinet

# 清理旧进程
sudo ./kill_pn_dev.sh

# 运行程序
sudo ./pn_dev -i eth0 -s rt-labs-dev
```

### 5.5 参数说明

| 参数 | 说明 |
|------|------|
| `-i eth0` | 指定网络接口 |
| `-s rt-labs-dev` | Profinet 设备名称（需与 TIA 配置一致） |
| `-p /tmp/pnet` | 数据文件存储路径（可选） |
| `-vvv` | 详细日志级别（可选） |

## 6. PLC 配置

### 6.1 导入 GSDML

1. 打开 TIA Portal
2. 选项 → 管理全局设备描述文件（GSD）
3. 导入 `GSDML-V2.43-RT-Labs-P-Net-Sample-App-20240530.xml`

### 6.2 添加设备

1. 硬件目录找到 `RT-Labs P-Net Sample App`
2. 添加到项目
3. 设备名称设置为 `rt-labs-dev`
4. 配置 I/O：
   - **Slot 1**：**DIO 8xLogicLevel**（IDM_32，8位输入+8位输出）

### 6.3 下载并启动

1. 下载硬件配置到 PLC
2. 启动 Profinet 通讯
3. 观察开发板串口输出

## 7. 调试与监控

### 7.1 串口输出格式

```
I1.0=0,I1.1=0,I1.2=0,I1.3=0,I1.4=0,I1.5=0,I1.6=0,I1.7=0 | Q1.0=1,Q1.1=0,Q1.2=0,Q1.3=0,Q1.4=0,Q1.5=0,Q1.6=0,Q1.7=0
```

### 7.2 后台运行

```bash
sudo ./pn_dev -i eth0 -s rt-labs-dev > /tmp/pn_dev.log 2>&1 &
tail -f /tmp/pn_dev.log
```

### 7.3 清理进程

```bash
sudo ./kill_pn_dev.sh
```

### 7.4 重置配置

```bash
# 删除运行时配置（恢复出厂设置）
rm -f /tmp/pnet/*.db
```

## 8. 注意事项

1. **设备名称必须一致**：`-s rt-labs-dev` 与 TIA 配置一致
2. **IP 分配**：首次启动为 0.0.0.0，PLC 连接后通过 DCP 分配
3. **实时内核**：已适配 Linux 6.1.118 RT
4. **调试串口**：使用 `/dev/ttyFIQ0`，失败时回退到 stdout
5. **静态链接**：`pn_dev` 已包含 p-net 协议栈，无需额外库文件
